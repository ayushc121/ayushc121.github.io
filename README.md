# Portfolio

A minimal project portfolio built for GitHub Pages. One source of truth: `projects.json`. The website and PDF both read from it.

---

## Quick start

1. Create a GitHub repo named `yourusername.github.io`
2. Drop all these files into it
3. Fill in `projects.json` (see schema below)
4. Add your images to `assets/`
5. Push to `main` — GitHub Actions builds the PDF and deploys the site automatically

Your site will be live at `https://yourusername.github.io`.

---

## Enable GitHub Pages (one-time setup)

In your repo on GitHub:

```
Settings → Pages → Source → GitHub Actions
```

That's it. The workflow handles everything else on every push.

## projects.json schema

```json
{
  "portfolio": {
    "name":     "Your Name",
    "tagline":  "One-line description shown under your name",
    "linkedin": "https://linkedin.com/in/yourhandle",
    "github":   "https://github.com/yourhandle",
    "email":    "you@example.com"
  },
  "projects": [
    {
      "id":          "unique-slug",
      "title":       "Project Title",
      "date":        "2024",
      "tags":        ["Python", "ROS", "Computer Vision"],
      "summary":     "One or two sentences shown on the project card.",
      "description": "Full description shown in the modal and PDF.\n\nUse double newlines for paragraphs.",
      "hero":        "assets/myproject/hero.jpg",
      "media": [
        { "type": "image",   "src": "assets/myproject/diagram.png",  "caption": "System diagram" },
        { "type": "youtube", "id":  "dQw4w9WgXcQ",                  "caption": "Demo video" },
        { "type": "vimeo",   "id":  "123456789",                     "caption": "Presentation" },
        { "type": "image",   "src": "assets/myproject/results.jpg",  "caption": "Results" }
      ],
      "links": {
        "github":  "https://github.com/you/project",
        "demo":    "https://yourproject.com",
        "video":   "https://youtube.com/watch?v=...",
        "paper":   "https://arxiv.org/abs/...",
        "report":  "https://..."
      }
    }
  ]
}
```

### Field reference

| Field | Required | Notes |
|-------|----------|-------|
| `portfolio.name` | Yes | Your name — appears in header, hero, and PDF footer |
| `portfolio.tagline` | No | Short line under your name in the hero |
| `portfolio.linkedin` | No | Shown as links in header, hero, footer |
| `portfolio.github` | No | Shown as link in hero |
| `portfolio.email` | No | Shown in footer |
| `project.id` | Yes | URL-safe slug, unique per project |
| `project.title` | Yes | |
| `project.date` | No | Free text — "2024", "Jan 2023", etc. |
| `project.tags` | No | Used for the filter bar |
| `project.summary` | No | Short text on the card (falls back to `description`) |
| `project.description` | No | Full text in modal + PDF; use `\n\n` for paragraphs |
| `project.hero` | No | Main image — path relative to repo root |
| `project.media` | No | Array of image/youtube/vimeo items (see above) |
| `project.links` | No | Any of: `github`, `demo`, `video`, `paper`, `report`, `website` |

### Media types

```json
{ "type": "image",   "src": "assets/...",  "caption": "optional" }
{ "type": "youtube", "id":  "VIDEO_ID",    "caption": "optional" }
{ "type": "vimeo",   "id":  "VIDEO_ID",    "caption": "optional" }
```

- **Images** — paths relative to repo root. Keep files under ~500 KB for fast loading.
- **YouTube / Vimeo** — embedded as iframes on the website; on the PDF they become a thumbnail + QR code that links to the video.
- Up to 4 media items appear in the PDF grid; all items appear on the website.

## Generating the PDF locally

```bash
pip install weasyprint requests
python generate_pdf.py
# Output: portfolio.pdf
```

Python 3.10+ required. WeasyPrint has additional system-level dependencies:

- **macOS:** `brew install pango cairo gdk-pixbuf libffi`
- **Ubuntu/Debian:** `sudo apt install libpango-1.0-0 libcairo2 libgdk-pixbuf-2.0-0`
- **Windows:** See [WeasyPrint docs](https://doc.courtbouillon.org/weasyprint/stable/first_steps.html#windows)

## File structure

```
your-repo/
├── index.html                   ← website
├── style.css
├── app.js
├── projects.json                ← your content (fill this in)
├── portfolio.pdf                ← auto-generated by CI
├── assets/
│   └── projectname/
│       ├── hero.jpg
│       └── diagram.png
├── generate_pdf.py              ← run locally or via CI
├── requirements.txt
└── .github/
    └── workflows/
        └── deploy.yml           ← builds PDF + deploys on every push to main
```

## Updating content

1. Edit `projects.json` (add/update/reorder projects)
2. Add any new images to `assets/`
3. `git add . && git commit -m "add project X" && git push`

GitHub Actions will regenerate the PDF and redeploy the site. Done.
